*conduit.txt*  Remote file editing and SSH integration for Vim 9

                                 CONDUIT.VIM                                 *conduit*

Conduit.vim provides seamless integration between your local Vim instance and
remote SSH sessions. It allows you to open files from a remote shell directly
in your local Vim, and perform background file transfers (get/put) with
progress notifications.

NOTE: This plugin requires Vim 9.1 or later with +job and +popupwin.

==============================================================================
1. Usage                                                    *conduit-usage*

To start using conduit, use the |:Conduit open| command to connect to a remote
host:
>
    :Conduit open user@example.com
<
This will open a terminal window running an SSH session. Within this session,
a special `lvim` command is available.

In the remote shell:
- `lvim file.txt` opens "file.txt" in your local Vim using netrw.
- `lvim split file.txt` opens it in a horizontal split.
- `lvim put file.txt` uploads a local "file.txt" to your remote CWD.
- `lvim get local_file.txt` downloads a remote file.

If `g:conduit_overwrite_vim` is set (default), `vim` is aliased to `lvim`. Use
`_vim` to run the actual vim binary on the remote host.

2. `:Conduit` Command                                      *conduit-commands*

:Conduit {subcommand} ...                                   *:Conduit*
        General command interface for Conduit operations.

        Subcommands:

:Conduit open [++SSHOPT ...] [++TERMOPT ...] [--] [user@]host[:port]
                                                            *:Conduit-open*
        Open a new conduit session to {host}. `++` options before the host
        are parsed as SSH flags or Vim terminal options. SSH options are
        forwarded to `ssh`. Terminal options include `++curwin`,
        `++vertical`, `++hidden`, `++norestore`, `++open`, `++close`,
        `++noclose`, `++rows=N`, `++cols=N`, `++eof=...`, `++api=...`,
        `++kill=...`, and `++opencmd=...`. Use `++--` to stop option parsing
        before the host if needed.

:Conduit deploy [++SSHOPT ...] [++TERMOPT ...] [--] [user@]host[:port]
                                                            *:Conduit-deploy*
        Deploy the conduit environment to {host} without opening a terminal.
        This is useful when you already have an SSH session elsewhere and
        only want to inject the remote `lvim` environment. On success, Conduit
        echoes and copies the remote `source` command.

:Conduit exit {connection-key}                              *:Conduit-exit*
        Cleanly close the conduit session for {connection-key}, removing
        remote temporary files and closing the reverse tunnel.

:Conduit disconnect {connection-key}                        *:Conduit-disconnect*
        Forcefully disconnect the named connection.

:Conduit source {connection-key}                            *:Conduit-source*
        Echo and copy the `source` command for the remote conduit RC file to
        the system clipboard.

:Conduit notifications {history|dismiss}                    *:Conduit-notifications*
        Show a history of conduit notifications or dismiss all active
        notifications.

:Conduit stop {get|put|*} {connection-key} {pattern}        *:Conduit-stop*
        Stop ongoing transfer jobs. `pattern` is fuzzily matched against the
        transfer's local and remote file names. Use `*` to stop all matching
        jobs for that connection and transfer type.

        Connection keys are resolved by exact key, by host name, or by a
        unique prefix. When multiple profiles are open for the same host, use
        the full profile key shown by completion or in notifications.

4. Configuration                                            *conduit-config*

g:conduit_default_split                                     *g:conduit_default_split*
        Default split mode when `lvim` is called without an operation.
        Default: "split"

g:conduit_overwrite_vim                                     *g:conduit_overwrite_vim*
        If true, aliases `vim` to `lvim` in the remote shell.
        Default: true

g:conduit_use_popup                                         *g:conduit_use_popup*
        If true, uses popup menus for file selection (e.g., when `lvim put`
        has multiple fuzzy matches).
        Default: false

g:conduit_put_max_depth                                     *g:conduit_put_max_depth*
        Maximum depth for fuzzy searching local files during `lvim put`.
        Default: 5

g:conduit_verbose                                           *g:conduit_verbose*
        Enable verbose logging of conduit operations.
        Default: false

g:conduit_sep                                               *g:conduit_sep*
        Separator used for internal communication.
        Default: ":::"

g:conduit_fallback_shell                                    *g:conduit_fallback_shell*
        Shell to use on the remote host if not specified in
        `g:conduit_host2shell`.
        Default: "bash"

g:conduit_host2shell                                        *g:conduit_host2shell*
        Dictionary mapping hostnames to remote shell paths.
        Default: {}

g:conduit_default_control_persist                           *g:conduit_default_control_persist*
        Default SSH ControlPersist value for manually managed multiplexing.
        Default: "60m"

g:conduit_host2sshoptions				    *g:conduit_host2sshoptions*
        Dictionary mapping hostnames to custom SSH flags. Flags are passed
        directly to ssh/scp/rsync command lines without validation. Use with
        caution; invalid or unsafe options may cause sessions to fail or
        behave unexpectedly.
        Default: {}

g:conduit_borderchars                                       *g:conduit_borderchars*
        List of 8 characters for popup borders.
        Default: Unicode if `+multi_byte`, otherwise ASCII fallbacks.
        `['Рћђ', 'Рћѓ', 'Рћђ', 'Рћѓ', 'РЋГ', 'РЋ«', 'РЋ»', 'РЋ░']` (Unicode)
        `['-', '|', '-', '|', '+', '+', '+', '+']` (ASCII)

g:conduit_success_timeout
	The number of milliseconds to persist success notifications.
        Default: 5000

g:conduit_failure_timeout
	The number of milliseconds to persist failure notifications.
        Default: 5000

5. Remote Command (lvim)                                    *conduit-lvim*

The `lvim` command is injected into the remote shell. It supports the
following syntax:

`lvim [op...] {args...}`

Operations:
The `lvim` command accepts one or more operations, followed by filenames or
arguments. Conduit identifies the longest prefix of arguments that match
available operations. Multi-word operations like `vert split` are recognized
automatically without requiring quotes.

Available {op}s are:
    - split
    - sp
    - vsplit
    - vsp
    - vert split
    - vertical split
    - tabe
    - tabedit
    - tabnew
    - tab split
    - tab sp
    - tab vsplit
    - tab vert split
    - tab vertical split
    - tab vsp
    - arga
    - argadd

{op}s can also be modifiers:
    - tab
    - vert
    - vertical
    - hor
    - horizontal
    - lefta
    - leftabove
    - abo
    - aboveleft
    - rightb
    - rightbelow
    - bel
    - belowright
    - to
    - topleft
    - bo
    - botright

Examples:
- `lvim vert split file.txt` Рђћ Opens file.txt in a vertical split (no quoting needed)
- `lvim tab split file.txt` Рђћ Opens file.txt in a new tab split
- `lvim belowright split file.txt` Рђћ Opens file.txt in a below split
- `lvim file.txt` Рђћ Opens file.txt using |g:conduit_default_split|
- `lvim -- file.txt` Рђћ Explicitly marks file.txt as a filename (useful if a filename matches an operation name)

File Transfer Operations:
- put {local_file} [{remote_file}]: Upload local {local_file} to remote {remote_file}. If `{remote_file}` is omitted, Conduit uses the remote shell's current directory.
- get {remote_file} [{local_file}]: Download remote {remote_file} to local {local_file}. If `{local_file}` is omitted, Conduit uses the local current directory.
- mget {pat}: Download remote files matching {pat}. If `{pat}` is not quoted, it will expand on the server and each individual file will be downloaded.
- mput {pat}: Upload local files matching {pat}. If no destination is provided, Conduit uses the remote shell's current directory. `{pat}` must be quoted, otherwise it will be expanded before by the shell before Conduit can access it.

If no operation is provided, |g:conduit_default_split| is used.

For `put`, if the local path is not found, conduit will attempt a fuzzy
search in the local CWD up to |g:conduit_put_max_depth|.

`Conduit open` and `Conduit deploy` also accept SSH options before the host
using a `++` prefix:

`:Conduit open ++J user1@host1 user2@host2`

This is translated to `ssh -J user1@host1 user2@host2` and applies to the
entire Conduit session.

When multiple profiles are open for the same host, Conduit uses a profile key
that appends a short hash to the host and port, for example
`user@host:22-1a2b3c4d5e6f`. Use that key with `:Conduit exit`,
`:Conduit disconnect`, `:Conduit source`, and `:Conduit stop`.

6. Notifications                                            *conduit-notifications*

Conduit uses a built-in notifier (based on popup windows) to show progress for
file transfers and status updates. If `+multi_byte` is enabled, it uses
fancy Unicode symbols (РюЊ, ├Ќ, Рєњ, etc.); otherwise, it falls back to
ASCII equivalents (=, x, ->, etc.).

g:notifier_maxwidth                                         *g:notifier_maxwidth*
        Maximum width of notification popups.
        Default: &columns / 2

g:notifier_overflow                                     *g:notifier_overflow*
        How notifications handle messages wider than `g:notifier_maxwidth`.
        Supported values:
                'wrap'      wrap onto multiple lines
                'truncate'  keep one line and trim with an ellipsis
                'carousel'  keep one line and scroll through the full message
        Default: 'carousel'
        In carousel mode, loading indicators such as spinners and progress bars
        stay fixed while the message text scrolls.

g:notifier_carousel_interval                 *g:notifier_carousel_interval*
        Delay in milliseconds between carousel frames.
        Default: 100

They automatically adapt to `+multi_byte` support.

You can view the notification history using

`:Conduit notifications history`

or dismiss all active notifications using

`:Conduit notifications dismiss`
